☎️ 이야기 5.3 패키지

DOS 통신 감성을 유지하면서 SSH-BBS에 연결하는 포터블 실행 환경

IYAGI 5.3 + DOSBox + SSH Bridge
View GitHub Repository

💡 무엇을 해결하나요?

이야기 5.3은 원래 모뎀 다이얼업 통신용 DOS 프로그램입니다. 이 프로젝트는 DOSBox 가상 모뎀과 Go bridge를 이용해, 기존 조작감은 유지하면서 SSH-BBS로 접속할 수 있게 패키징합니다.

목표는 단순합니다: 설치 의존성을 최소화하고, 한 번 빌드한 결과물을 Windows, macOS, Linux에서 바로 실행 가능한 형태로 배포합니다.

🔌 연결 구조

이야기 5.3 (DOS)
  -> DOSBox COM4 (AT 명령 전달)
  -> bridge (ATDT 파싱 + SSH 연결)
  -> SSH 서버/SSH-BBS (예: 127.0.0.1:40000)
  • SSH_AUTH_MODE=bbs 기본: 키 없이 SSH 세션을 열고 BBS 내부 로그인 사용
  • SSH_AUTH_MODE=key 선택: 기존 SSH 공개키 인증 방식 사용
  • IYAGI 다이얼 문자열의 대상(ATDT host:port)을 bridge가 그대로 파싱해 SSH 대상으로 사용합니다
  • BRIDGE_PORT는 IYAGI가 접속할 로컬 브리지 포트입니다 (기본 2323)
  • ZMODEM 기반 업/다운로드 경로를 런처에서 일관되게 유지

⚙️ 필수 설정 예시

# bridge가 듣는 로컬 포트 (IYAGI가 먼저 이 포트로 접속)
BRIDGE_PORT=2323

# 기본값(미입력 ATDT를 대비한 안내용)
IYAGI_HOST=127.0.0.1
IYAGI_SSH_PORT=40000
IYAGI_USER=your-user

# SSH-BBS라면 보통 bbs 모드 권장
SSH_AUTH_MODE=bbs

핵심: 이제 bridge가 ATDT127.0.0.1:40000 같은 다이얼 문자열을 직접 파싱하여 해당 SSH 서버(127.0.0.1:40000)로 연결합니다.

✨ 주요 기능

🧱 포터블 패키지

Linux(AppImage), Windows(Installer), macOS(DMG)로 배포 가능한 구조를 제공합니다.

🖥️ DOS 감성 유지

이야기 5.3 원본 실행 흐름을 유지하면서 SSH-BBS로 연결합니다.

🔧 자동 설정 패치

I.CNF, I.TEL 자동 패치로 브리지 접속 항목/경로를 맞춰줍니다.

🧹 런타임 정리

브리지 프로세스 시작/종료를 런처에서 안전하게 관리합니다.

🌈 DOSBox-X 최적화

스케일러/메뉴바/창 크기 설정을 프로젝트 환경에 맞춰 조정했습니다.

🚀 빠른 시작

1) Linux AppImage 빌드

bash tools/build-linux.sh

2) 직접 실행(개발/테스트)

bash tools/run-direct.sh

3) AppImage 실행

bash tools/run-appimage.sh

상세 사용법은 한국어 매뉴얼을 참고하세요.
