# 이야기 5.3 사용자 매뉴얼 (이 프로젝트용)

이 문서는 `DOS + 이야기 5.3`를 이 저장소의 패키지(직접 실행/AppImage)로 사용하는 방법을 정리한 한국어 매뉴얼입니다.

## 1) 개요

- `이야기 5.3`는 원래 모뎀(AT 명령) 기반 통신 프로그램입니다.
- 이 프로젝트는 모뎀 대신 다음 경로로 SSH-BBS에 연결합니다.
  - `이야기 5.3` -> `DOSBox 가상 모뎀` -> `Go bridge` -> `SSH 서버(또는 SSH-BBS)`
- 기본 구성은 `SSH_AUTH_MODE=bbs`이며, SSH 키 없이 세션을 열고 BBS 내부 로그인(ID/PW)을 사용하는 시나리오를 기본으로 둡니다.

## 2) 실행 방법

### 2-1. 직접 실행 (개발/테스트)

```bash
bash tools/run-direct.sh
```

### 2-2. AppImage 실행

```bash
bash tools/run-appimage.sh
```

또는 AppImage 직접 실행:

```bash
./dist/IYAGI-linux-x86_64.AppImage
```

## 3) 첫 실행 시 설정 파일

실행 모드에 따라 `.env` 위치가 다릅니다.

- 직접 실행(`tools/run-direct.sh`) 기본값: `iyagi-data/.env` (저장소 루트 기준)
- AppImage 직접 실행 기본값:
  - 설정: `${XDG_CONFIG_HOME:-~/.config}/iyagi-terminal/.env`
  - 데이터: `${XDG_DATA_HOME:-~/.local/share}/iyagi-terminal/`
- Windows 패키지(`launch.bat`) 기본값:
  - 설정: 설치/압축 해제 폴더의 `.env`
  - 예: `C:\Users\<사용자>\Downloads\IYAGI\.env`
- macOS `.app` 런처(`Contents/MacOS/launcher`) 기본값:
  - 설정: `~/Library/Application Support/IYAGI/.env`
  - 데이터: `~/Library/Application Support/IYAGI/downloads/`
- 공통: `USER_DATA_ROOT`를 지정하면 해당 경로를 우선 사용합니다.

런처의 값 해석 우선순위:

1. 셸에서 export한 환경변수
2. `.env`
3. 런처 기본값(`${VAR:-default}`)

현재 접속 모델은 `ATDT <host>:<port>` 직접 다이얼이므로, `IYAGI_HOST`/`IYAGI_USER`는 필수값이 아니라 기본 힌트용입니다.

```dotenv
IYAGI_HOST=접속할_서버
IYAGI_USER=ssh_접속_계정
BRIDGE_PORT=2323
SSH_AUTH_MODE=bbs
```

주요 옵션:

- `SSH_AUTH_MODE=bbs`: 기본값. 키 생성 없이 SSH-BBS 스타일 접속.
- `SSH_AUTH_MODE=key`: 전통 SSH 키 인증 사용(이 경우에만 `iyagi-data/keys` 사용).
- `DOSBOX_X_SCALER`: 화면 보간/스케일러
  - 예: `hq2x forced` (부드럽게), `normal2x forced` (픽셀 선명)

## 4) 접속 절차 (이 프로젝트 기준)

1. 런처가 `bridge`를 먼저 실행합니다. (`127.0.0.1:2323` 대기)
2. DOSBox에서 이야기 5.3가 시작됩니다.
3. 이야기에서 `ATDT호스트:포트`를 입력해 다이얼합니다. (예: `ATDT127.0.0.1:40000`)
4. bridge가 `ATDT`의 호스트/포트를 파싱해 해당 SSH 대상으로 연결합니다.

## 5) 전화번호부/통신 설정

이 프로젝트는 빌드 시 `scripts/configure_iyagi.py`로 이야기 설정 파일을 자동 패치합니다.

- 과거 `127.0.0.1:2323` 고정 항목은 제거
- COM4 -> bridge(`BRIDGE_PORT`) 경로 사용
- 다운로드 경로를 패키지 구조에 맞게 조정

수동 조정이 필요하면 이야기 내부의 전화번호부/통신설정 메뉴에서 다음 개념을 확인하세요.

- 통신 속도(BPS)
- 패리티/데이터비트/스톱비트
- Flow Control(XON/XOFF, RTS/CTS)

## 6) 파일 전송 (ZMODEM)

이야기 5.3는 ZMODEM 전송을 지원합니다.

- 업로드: 이야기의 파일전송 메뉴에서 업로드 실행
- 다운로드: 이야기의 파일전송 메뉴에서 다운로드 실행
- 자동 다운로드/프로토콜 동작은 상대(BBS) 측 설정에 따라 달라질 수 있습니다.

수신 파일 저장 위치:

- 직접 실행 기본값: `iyagi-data/downloads/`
- AppImage 기본값: `${XDG_DATA_HOME:-~/.local/share}/iyagi-terminal/downloads/`

## 7) 자주 쓰는 조작

원본 도움말(`I.HLP`) 기준으로 F1 도움말 체계가 제공됩니다.

- `F1`: 도움말/현재 항목 안내
- 통신, 캡처, 전송, 스크립트 등은 이야기 내부 메뉴에서 접근 가능

주의: 배포본/키맵 파일(`KEY.DEF`)에 따라 일부 단축키 체계가 달라질 수 있으므로, 실제 사용 중에는 `F1` 도움말을 우선 기준으로 보세요.

## 8) 스크립트 기능

원본 도움말과 `NEW.DOC`에 따르면 이야기 5.3는 스크립트(자동화) 기능을 제공합니다.

- 대기 문자열(`Wait_str`)
- 문자열 송신(`Send_str`)
- 분기(`If/Else`)
- 지연(`Delay`)
- 업/다운로드 호출

자동 로그인/자동 메뉴 진입 같은 반복 작업을 줄일 때 유용합니다.

## 9) 문제 해결

### 9-1. 키 폴더가 보이는데 필요한가?

- 기본 `SSH_AUTH_MODE=bbs`에서는 필요 없습니다.
- `SSH_AUTH_MODE=key`일 때만 `iyagi-data/keys`를 사용합니다.

### 9-2. 창이 너무 크거나 일부만 채워짐

- `tools/run-direct.sh`와 `tools/run-appimage.sh`를 같은 `USER_DATA_ROOT`로 맞춰 실행하세요.
- `.env`에서 스케일러를 조정하세요.
  - 선명: `DOSBOX_X_SCALER="normal2x forced"`
  - 부드러움: `DOSBOX_X_SCALER="hq2x forced"`

### 9-3. 메뉴바(상단 DOSBox-X 메뉴) 숨기기

현재 런처는 DOSBox-X 실행 시 메뉴를 숨기도록 설정되어 있습니다.
